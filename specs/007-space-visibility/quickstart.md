# Quickstart: Space Visibility Indicators

**Feature**: 007-space-visibility  
**Date**: 2026-02-26

## Prerequisites

- Node.js 20+, pnpm >= 9
- Running Alkemio platform instance with spaces that have mixed privacy settings
- Valid Alkemio credentials (email/password)
- Repository on branch `007-space-visibility`

## Setup

```bash
# Install dependencies
cd server && pnpm install
cd ../frontend && pnpm install
cd ..
```

## Development

```bash
# Terminal 1: Start BFF server
cd server
cp .env.example .env  # Edit ALKEMIO_GRAPHQL_ENDPOINT if needed
pnpm run dev

# Terminal 2: Start frontend
cd frontend
pnpm run dev
```

Open http://localhost:5173 in your browser.

## Testing the Feature

### 1. Verify Data Pipeline

After logging in and selecting spaces:

```bash
# Check that privacyMode appears in the API response
curl -s http://localhost:4000/api/graph/generate \
  -H "Authorization: Bearer <your-session-token>" \
  -H "Content-Type: application/json" \
  -d '{"spaceIds": ["your-space-id"]}' | \
  jq '.nodes[] | select(.type | startswith("SPACE")) | {displayName, type, privacyMode}'
```

Expected output:
```json
{ "displayName": "My Public Space", "type": "SPACE_L0", "privacyMode": "PUBLIC" }
{ "displayName": "Private Subspace", "type": "SPACE_L1", "privacyMode": "PRIVATE" }
```

### 2. Visual Verification Checklist

| # | Check | Expected Result |
|---|-------|-----------------|
| 1 | Load graph with mixed public/private spaces | Each space node shows lock (🔒) or unlock (🔓) icon |
| 2 | Zoom in on a space node | Icon remains legible and proportional |
| 3 | Zoom out to overview | Icons are still visible (small but present) |
| 4 | Select a space node | Icon remains visible during selection highlight |
| 5 | Hover over a space node | Icon doesn't interfere with hover card |
| 6 | Check control panel "Visibility" section | Two checkboxes: "Public (N)" and "Private (N)" |
| 7 | Uncheck "Public" | Public spaces + orphaned connections disappear |
| 8 | Re-check "Public", uncheck "Private" | Private spaces disappear, public reappear |
| 9 | Toggle both off | All spaces hidden |
| 10 | Toggle both on | All spaces restored, positions unchanged |
| 11 | Uncheck "Spaces" entity filter | Visibility section becomes disabled |
| 12 | Check legend section | "Visibility" group with public/private icon explanations |

### 3. Edge Case Testing

| # | Scenario | Expected |
|---|----------|----------|
| 1 | All spaces are public | "Private (0)" shown, no lock icons visible |
| 2 | All spaces are private | "Public (0)" shown, all spaces show lock |
| 3 | Mix of L0 public + L1 private subspaces | Each level shows independent correct icon |
| 4 | User connected to both public and private spaces | User remains visible when either filter is on |
| 5 | Activity pulse enabled + visibility icons | Both animate/display without conflict |
